All About GitHub

Question and Answer

  • Q: What is a version control system?
  • A: A software tool that tracks changes in files by keeping track of modifacations done in the code.
  • Q: What is a Git?
  • A: A distributed version control system that tracks changes in any set of computer files.
  • Q: What is GitHub?
  • A: A hosting service for software development and version control using Git.
  • Q: What is the difference between Git and Github?
  • A: GitHub is a software tool that utilizes Git.
  • Q: Who started GitHub and how was it started?
  • A: It was developed by Chris Wanstrath, P.J. Hyett, Tom Preston-Werner and Scott Chacon. It was created to provide developers the option to host code securly and manage commits to code in proper manner.
  • Q: What company owns it now?
  • A: Microsoft.
  • Q: How much does a GitHub account cost?
  • A: It is free for individuals and organizations. There are options that offer additional security/options for $4 and $21.
  • Q: What is the octocat?
  • A: The octocat is GitHub's logo and mascot.

Git/GitHub terms

  • Repository: A location in the GitHub platform where the files and codes corresponding to the projects and their respective versions as a part of revision history is stored, managed, and used.
  • Commit: An individual change to a file (or set of files).
  • Fork: A copy of a repository
  • Push: Send your committed changes to a remote repository.
  • Pull Requests: A request to the maintainer of a repository to pull in some code.
  • Workflows: A configurable automated process made up of one or more jobs
  • Issues: Issues let you track your work on GitHub, where development happens.
  • Raw Button: Opens files in raw form, meaning that the HTML formatting dissapears.
  • Blame Button: Describes the last modifacation to each line of a file, which generally displays the revision, author and time.
