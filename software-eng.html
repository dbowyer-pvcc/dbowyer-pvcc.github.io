Term Definition
Software Engineering A branch of computer science that uses engineering principles to develop computer programs and operating systems.
Integrated Development Environment (IDE) software application that helps programmers develop software code efficiently.

Software Life Cycle
  • Requirements Analysis: An element of project managment that helps ensure clarity, completeness and relevance. The goal is to define expectations for a project.
  • Design: The creation of a plan or a convention for the construction of an object, system or measurable human interaction.
  • Implementation: A realization of a technical specification or algorithm as a program, software component, or other computer system through computer programming and development.
  • Testing: Helps programmer remove bugs and ensure that program functions as intended.
Modular Implementation Groups related sets of functions together into a modual.
Unified Modeling Language (UML) A general purpose modeling language. The main aim of UML is to define a standard way to visualize the way a system has been designed.
Use Case A software and system engieneering term that describes how a user uses a system to accomplish a particular goal.
Use Case Diagram A graphical depiction of a user's possible interactions with a system.
Data Relationships
  • One-To-One: A relationship where a record in one entity (table) is associated with exactly one record in another entity.
  • One-To-Many: Means thst one row in a table (called the parent table) can have a relationship with many rows in another table (child table).
  • Many-To-Many: A type of cardinality that refers to the relationship between two entities.
Design Pattern Formalizes best practices that the programmer can use to solve common problems with designing an application.
Software Qulaity Assurance (SQA) A means and practice of monitoring the software engieneering processes and methods used in a project to ensure proper quality of the software.
Software Testing
  • Glass Box Testing: A method of software testing that tests internal structure or workings of an application, as opposed to its functionality.
  • Black Box Testing: A method of software testing that examines the functionality of an application without peering into its internal structures or workings.
  • Beta Testing: A phase in testing in which the software is tested by a larger group of users, typically outside of the organization that developed it.
  • Alpha Testing: A type of software testing performed to identify bugs before realizing the product to real users or to the public.
Documentation
  • User Documentation: Any form of documentation intended for the end-user of a product or a service.
  • System Documentation: Represents documents that describe the system itself and its parts. It includes requirements documents, design decisions, archetecture descriptions, program source code, and help guides.
  • Technical Documentation: Documentation that describes how a product or service operates.
Intellectual Property Law
  • Software License: A document that states the rights of the developer and user of a piece of software.
  • Software Disclaimer: Lets users know that there are limits regarding what developers can be held responsible for.
